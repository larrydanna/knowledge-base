# How We Got Here — The Origin of This Knowledge Base

**Date:** 2026-04-14  
**Author:** Larry D. Collins Jr.

---

## The Starting Point

This knowledge base began during the onboarding of an old but clean Windows 10 laptop
for AI learning and development. There was no master plan — just a desire to learn and
a habit of reaching for good information.

---

## The Workflow That Started It

Whenever a new topic came up, the process was simple:

1. Search Google for the topic
2. Click **"Dive Deep with AI"** in the search results
3. Copy and paste the returned markdown into a local file in this directory

This produced useful notes quickly, but without structure — just a loose collection of
markdown files with no consistent naming, no index, and no clear portability story.

---

## The Turning Point

The need for a more durable solution became clear: something **portable**, something that
could follow across machines and grow over time. A GitHub repository was the natural fit.

Rather than designing the structure from scratch, GitHub Copilot was brought in — in
**plan mode** — with a prompt to generate a clean, clear practice for managing a
personal knowledge base. The plan was reviewed, refined, and then implemented.

---

## What That Session Produced

That Copilot planning session established:

- A consistent **file naming convention** (`kebab-topic-YYMMDDa.md`)
- A **folder structure** (`environment/`, `engineering/`, `meta/`, `prompts/`)
- A reusable **entry template** with TL;DR, body, and sources
- A **master index** (`INDEX.md`) for quick AI session priming
- A **shared context file** (`context-base.md`) for identity and preferences
- A set of **reusable AI prompts** for common tasks
- A **git workflow** — simple, no-branch, commit-and-push

---

## The Lesson Learned

> **Ad-hoc learning accumulates fast. Structure it early.**

The copy-paste approach was fine for capturing individual notes, but without naming
conventions, an index, or a portability strategy, those notes were hard to reuse —
especially across machines or AI sessions.

Bringing in a planning tool (Copilot in plan mode) to design the structure — rather than
improvising it — produced a system that is clean, consistent, and extensible from the start.

---

## The Meta-Lesson

This origin story is itself an example of the workflow it describes:

- A real session happened
- Notes were captured
- Structure was applied
- The result is now documented and preserved for future reference

That's the loop this knowledge base is designed to sustain.
